III. R Basics

Let’s jump right into some basics of R. You can enter these commands directly in the console (select enter after each command), or you can enter them in the script, then select Run All.

At a very basic level, R is a calculator.

#Enter the following in your R console and hit enter.
9 + 8
## [1] 17

It is often valuable to assign numerical values as variables. The assignment operator in R is “<-”.

#Enter the following in your R console and hit enter.
x <- 9 + 8

While this code runs, it doesn’t appear like anything happened, but in reality you just assigned x the value of 9 + 8. If you type x into your console and hit enter, then R will return the value that x represents.

#Enter the following in your R console and hit enter.
x
## [1] 17

Vectors of multiple values can be creating using the concantenate function c().

#Enter the following in your R console and hit enter.
y <- c(9,8,7,6,5,4,3,2,1)
y
## [1] 9 8 7 6 5 4 3 2 1

Now we can convert this vector into a dataframe. This is like a dataset that can be used by various programs for analysis. This code will name the dataframe “df” using the data.frame() function. After running this code, the dataframe, df, will appear in the environment window, where this data can be clicked and viewed. Note, df is a name for the dataframe that is determined by the user, not a program specific name or function.

#Enter the following in your R console and hit enter.
df <- data.frame(y)
df

It is worth noting that if you look to the environment window in the upper right pane that df is now listed as a dataframe. If you click on the df dataframe in the evironment window, it will open up in the script pane (upper left pane) and you will be able to see the data that comprises the dataframe.

Now, suppose we had another vector of data, z, that we wanted to add as a column to the df dataframe.

#Enter the following in your R console and hit enter.
z <- c(1,2,3,4,5,6,7,8,9)

df <- data.frame(df,z)
df

Suppose you want to rename the columns in df “x1” and “x2”, you can use the names() command.

#Enter the following in your R console and hit enter.
names(df) <- c("x1", "x2")
df

After executing these commands, click on the df dataframe in the environment pane to see that the variable names have been changed.

Now we can do some basic data manipulation. Here we are going to multiply the two columns together to create a third column within the dataframe. There are many ways to do this, but one direct method is to call the variables in the dataset using the dataframe name “df” with the “$” and the variable name, then apply the data manipulation. Here we are going to make a new variable “x3” by multiplying “x1” and “x2” together. In terms of basic math operators, R recognizes addition (+), subraction (-), multiplication (*), and division (/) with standard character operators (i.e. the same as you would use in Excel for example).

#Enter the following in your R console and hit enter.
df$x3 <- df$x1*df$x2
df

IV. Making an R Script

Up until this point, you have been entering commands directly into the R console, seeing the immediate result of the code. However, the practical reality is that you want to keep track of the code and the program that you are creating, this is where the r script comes into play. In order to open an r script, you can click the plus sign with the white square icon in the upper left corner of R studio, and select R script. Alternatively, you can select File > New File > R Script from the menu bar above. Once selected, a fourth box will open in upper left pane where you will be able to type in code.

To give this a try, let’s load all of the code that we have entered thus far in this tutorial into an r script.

Thus far in this tutorial, you have excecuting the following code:

9 + 8
x <- 9 + 8
x

y <- c(9,8,7,6,5,4,3,2,1)
y

df <- data.frame(y)

z <- c(1,2,3,4,5,6,7,8,9)

Copy and paste this block of code into the R Script pane. Now that the code is in the script pane, highlight the entire block of code (or select all) and click the “Run” button at the top of the script pane. You will see in the console (lower left pane) that all of your code ran as a block (rather than as individual lines like before), and your script is untouched. The script allows you to preserve your code and run the code in full (or in pieces based on your selection/highlighting). Your script can also be saved as a file to be opened later.

One key to programming code is keeping track of what you are doing. One of the features in R is the ability to add comments to your code. In order to add a comment to a code, you just have to begin the line with a “#”. This character tells R not to read the text that follows this “#”. Thus you can, and should use comments frequently throughout the code to remember and explain what different blocks of code are doing. Your future self will always be thankful to your present self for good code commenting.

For example, let’s look at a snippet of code from above and add some comments to explain what we are doing.

# This block of code demonstrates how R can be used like a calculator and shows how R uses an assignment operator to assign values to variables.
9 + 8
x <- 9 + 8
x

# This is using the concatenate function c() to build a vector of data.
y <- c(9,8,7,6,5,4,3,2,1)
y

# As you can see, once you begin a line with a "#", you can write any text you want and it will not impact the program.

VI. Data Visualization Introduction

The ggplot Package

As an introduction to visualizing data, we are going to use the “ggplot2” package.

R works by leveraging various user created packages. A nice analogy (not to bring you back to the horror ACT/SAT type questions) is that apps to a smartphone are like packages to R. R has all kinds of packages (many of which are created by individual users and made publicly available for free) that serve a variety of functions. In order to use a package, you must first install the package, then you need to call the package using the library command.

In order to familiarize you with this concept and some basic data visualization, we will use the ggplot package, which is arguably one of the most commonly used packages in R. If this is your first time using ggplot you need to first install it with the below install.packages() command. You will only need to install a package once.

install.packages("ggplot2")

However, if you are going to use a package, you must call it in your program using the library commeand. This must be done in your code anytime you use a package (typically, these library commands will be laid out at the beginning of a script).

library(ggplot2)

The ggplot2 package is designed to build graphs in layers, adding one visuals ane features on top of one another. The below code will set up the base graph and add features one at a time.

The basic ggplot command requires the call of a dataframe and aesthetics information, most notably the x and y axis variables (at the basic level). The ggplot command also requires the use of a geom function, which specifies the type of desired visual. For example, a line graph is created by using the geom_line() function. In this case, we are going to build a simple plot of variables x1 and x3 from our original dataframe df. Specifically, we want to plot x1 on the horizontal (x) axis, and x3 on the vertical (y) axis.

ggplot(data=df, aes(x=x1, y=x3)) +
  geom_line()

Now let’s add some layers to change the titles and axis labels. Labels can be added to the x and y axis using the xlab() and ylab() commands, and the graph title can be changed using ggtitle().

It is important to note that you need to end each of the lines with a + sign to indicate that this is all one block of code. The only exception of course is the last line in the ggplot code. This line should not have a plus sign as that is the end of the code.

ggplot(data=df, aes(x=x1, y=x3)) +
  geom_line() +
  xlab("X Axis") +
  ylab("Y Axis") +
  ggtitle("The Title of the Figure")

The ggplot package in R has a standard output appearence by default. This appearence can be completely customized based on the user’s needs. However, ggplot also has some built in themes to speed up the process. For example, let’s simplify this graph down by adding theme_classic() to the first line of the code.

ggplot(data=df, aes(x=x1, y=x3)) + theme_classic() +
  geom_line() +
  xlab("X Axis") +
  ylab("Y Axis") +
  ggtitle("The Title of the Figure")

Note, when you enter “theme” into R, you may notice that an autofill box pops up in R. This autofill box will provide all of the various preloaded themes in R. Take a few minutes and explore some of the themes in ggplot.

Chart Types & ggplot

The beauty of ggplot is that it is quite simple to change the type of a chart. All of the ggplot chart types are coded as geoms. For example, above we built a line chart using the command geom_line(). What if we wanted to make a simple scatterplot? For this we would just use the geom_point command. Let’s take our previous block of code and substitute geom_point() for geom_line().

ggplot(data=df, aes(x=x1, y=x3)) + theme_classic() +
  geom_point() +
  xlab("X Axis") +
  ylab("Y Axis") +
  ggtitle("The Title of the Figure")

What if I wanted to graph the points on the plot and combine that with the line graph? Do you remember that this package is based on the idea of layering? All we have to do is build the basic point chart and layer the line chart on top of it by adding the geom_line() command after the geom_point() command.

ggplot(data=df, aes(x=x1, y=x3)) + theme_classic() +
  geom_point() +
  geom_line() +
  xlab("X Axis") +
  ylab("Y Axis") +
  ggtitle("The Title of the Figure")

Sometimes a graph needs a little color to make it more appealing. Let’s change the points to a dark green color and the line to a gray using the color= command. Also, let’s make the points a little larger for emphasis using the size= command.

ggplot(data=df, aes(x=x1, y=x3)) + theme_classic() +
  geom_point(color="darkgreen", size=3) +
  geom_line(color="gray50") +
  xlab("X Axis") +
  ylab("Y Axis") +
  ggtitle("The Title of the Figure")

What if we wanted to make a bar chart? All we need to do is use the command geom_bar()! However, in this case, we need to be a little more specific given the options available in the geom_bar() command. The geom_bar command allows for the plotting of various statistics within the data set, to plot the actual data as is, you need to use the stat=identity option in geom_bar(). Let’s plot our simple dataset as a bar chart. Let’s also make the bars in our barchart a dark green color as well using the fill= command. Note this command is slightly different for bar charts. The color= command controls the color of the bar outline, while the fill= command determines the color inside the bar.

ggplot(data=df, aes(x=x1, y=x3)) + theme_classic() +
  geom_bar(stat="identity", color="gray50", fill="darkgreen") +
  xlab("X Axis") +
  ylab("Y Axis") +
  ggtitle("The Title of the Figure")

VII. Fun with Themes

One of the advantages of R is that it is a free, open-source program. This means that anyone can build R packages and make them openly available. We will dig into this idea in more depth with future lab tutorials, but for now let’s explore the ggthemes package.

We already highlighted that ggplot has a series of built in themes for quick styling of graphs/figures. You may be wondering if there are other types of preformatted themes available? The answer is yes! The ggthemes package contains more options and styles for preformatted themes. To make it even more interesting, ggthemes even has formats to match the style guides of well known publications such as The Economist magazine and FiveThirtyEight to name a few.

Let’s explore this package a little more.

First, we of course have to install the package.

install.packages("ggthemes")

Second, we have to call the package using the library

library(ggthemes)

Now that our package is installed and called with the library, now let’s try to implement this package by changing the themes on our bar chart. Like before, all we need to do is just add theme_ in the first line of code with our theme of interest. First, let’s convert our bar chart to the fivethirtyeight style using theme_fivethirtyeight(). Also, to harness the full power of the theme, let’s also drop our color specifications for the time being in the geom_bar command.

ggplot(data=df, aes(x=x1, y=x3)) + theme_fivethirtyeight() +
  geom_bar(stat="identity") +
  xlab("X Axis") +
  ylab("Y Axis") +
  ggtitle("Figure: theme_fivethirtyeight style")

How about we make our figure match the style of The Economist magazine?

ggplot(data=df, aes(x=x1, y=x3)) + theme_economist() +
  geom_bar(stat="identity") +
  xlab("X Axis") +
  ylab("Y Axis") +
  ggtitle("Figure: theme_economist style")

How about we convert this to a Wall Street Journal style?

ggplot(data=df, aes(x=x1, y=x3)) + theme_wsj() +
  geom_bar(stat="identity") +
  xlab("X Axis") +
  ylab("Y Axis") +
  ggtitle("Figure: theme_wsj style")

For more perspective on the themes available in the ggthemes package, please view the ggthemes reference page.
